Validation Walkthrough — Snapshot + Staging

Status Runbook Sample Report

Step 1 — Create a snapshot (WSL)

Copy
cd ~/Recon-automation-Bug-bounty-stack
chmod +x scripts/*.sh
bash scripts/snapshot_env.sh

This prints a path like snapshots/2025MMDD-HHMMSS (and .tar.gz).

Step 2 — Staging dry‑run (no network impact)

Copy
cd ~/Recon-automation-Bug-bounty-stack
bash scripts/run_staging.sh
RUN_DIR=$(ls -1dt env/staging/output/* | head -1)
echo "$RUN_DIR"
cat "$RUN_DIR/reports/summary.md"

Step 3 — Restore a snapshot (optional)

Copy
cd ~/Recon-automation-Bug-bounty-stack
bash scripts/restore_snapshot.sh snapshots/2025MMDD-HHMMSS

Step 4 — View proof in Windows browser

Copy
powershell.exe -NoProfile -Command "Start-Process '\\wsl$\\Ubuntu\\home\\ubuntu\\Recon-automation-Bug-bounty-stack\\docs\\sample_report.html'"
powershell.exe -NoProfile -Command "Start-Process '\\wsl$\\Ubuntu\\home\\ubuntu\\Recon-automation-Bug-bounty-stack\\docs\\status.html'"

WSL usually has no GUI; use Windows to open local HTML.
